Account Editor Delayed Master Flow — Shopify Flow Automation Reference

3PL Integrations  •  Last updated: April 2026

This is the single canonical Shopify Flow used across all 3PL integrations that require delayed payment capture with Account Editor. It creates an editing window for customers, then automatically captures payment and tags the order — triggering your 3PL to download the finalized order with all edits included.

📝

One Flow for all integrations. This Master Flow handles every scenario — different sales channels, fraud risk levels, restricted orders, and normal orders with a configurable grace period. Import it once, configure the timing, and it works for any supported 3PL.

⚡ The Complete Flow

This is the full Account Editor Delayed Master Flow with all conditions, the grace period wait, order tagging, and payment capture.

🔄 Account Editor Delayed Master Flow
✅ 7-Step Automation
1Trigger — Order Created

This workflow starts when a new order is created in your Shopify store.

2Condition 1 — Sales Channel Check

The Flow checks whether the order came from the Online Store channel.

  • If NOT from Online Store (POS, draft order, other channels) → skip to Step 7 — capture payment immediately. No wait, no tag.
  • If from Online Store → continue to Step 3.
3Condition 2 — Fraud Risk Check

Shopify’s fraud analysis checks the order risk level.

  • If risk level is HIGH → skip to Step 7 — capture payment immediately. No wait, no tag.
  • If risk is not HIGH (low or medium) → continue to Step 4.
4Condition 3 — Restricted Tag Check

The Flow checks whether the order is tagged restricted.

  • If tagged “restricted” → skip to Step 7 — capture payment immediately. No wait, no tag.
  • If not restricted → continue to Step 5.
5Wait — Grace Period

The Flow waits for 15 minutes (configurable). This is the editing window where the customer or merchant can modify the order in Account Editor — add/remove items, change quantities, update the address.

During this wait: Payment stays Authorized. Your 3PL does not see the order. All edits happen freely.

6Add Order Tag — processed_by_AE

The Flow adds the tag processed_by_AE to the order. This tag identifies orders that went through the full Account Editor editing flow.

Required by: Black Bear Fulfilment, ACT Logistics, Future Fulfilment. Harmless for all other 3PLs — the tag is simply ignored.

7Capture Payment

The Flow captures the authorized payment. Order status changes from Authorized to Paid. Your 3PL now downloads the finalized order with all edits included.

📊 Branch Summary

Every order follows one of these four paths:

Scenario Wait? Tagged? Action
Order from other channels (POS, draft, B2B) No No Capture payment immediately
Online Store + HIGH fraud risk No No Capture payment immediately
Online Store + tagged “restricted” No No Capture payment immediately
Online Store + normal order (low/medium risk, not restricted) Yes (15 min) Yes (processed_by_AE) Wait → Tag → Capture
⚙️ Configuration Notes
⏱️ Wait Time (Grace Period)
📝 Configurable per merchant

The default wait time is 15 minutes. You can adjust this in the Shopify Flow editor to match your editing deadline. Recommended range: 15 minutes to 2 hours.

⚠️

Keep timings in sync: The editing deadline shown to the customer in Account Editor should match (or be slightly shorter than) the Flow wait time. For example: Flow = 30 min, Editing Deadline = 25–30 min. Never set the editing deadline longer than the Flow timing.

🏷️ The processed_by_AE Tag
📝 Why it’s added

The tag is added after the wait but before capture. It serves two purposes:

  • Identifies edited orders — Distinguishes orders that went through the full editing window from immediately-captured orders (POS, high risk, restricted)
  • Required by some 3PLs — Black Bear Fulfilment, ACT Logistics, and Future Fulfilment use this tag to identify processed orders. All other 3PLs safely ignore it
⚡ Why Some Orders Capture Immediately
📝 No wait, no tag
  • Non-Online Store orders (POS, draft, B2B) — These are merchant-initiated. No customer editing window is expected
  • HIGH fraud risk orders — These need manual review. Capturing immediately lets you process or cancel them quickly
  • Restricted orders — A business-specific flag indicating the order should not be delayed (e.g., regulated products, compliance requirements)
📋 Prerequisites
💳 Payment Capture Must Be Set to Manual
✅ Required before importing the Flow

Before importing this Flow, ensure Shopify is set to manually capture payment. If payment is captured automatically at checkout, the Flow has nothing to delay — orders will be captured and downloaded by your 3PL before any editing happens.

  1. 1Go to Shopify Admin → Settings → Payments
  2. 2Set Payment capture method to “Manually capture payment for orders”
  3. 3Click Save
⚠️

7-day limit: Shopify Payments authorizations expire after 7 days. Keep your grace period well within this window. If the Flow fails or is deactivated, monitor for “Authorized” orders older than your grace period.

📥 How to Import the Flow
⚡ Import into Shopify Flow
✅ Step-by-Step
  1. 1Download the Account Editor Delayed Master Flow file using the button below
  2. 2In Shopify Admin, go to Apps → Shopify Flow
  3. 3Click Import and select the downloaded .flow file
  4. 4Review the Flow — adjust the wait time in Step 5 to match your editing deadline (default: 15 minutes)
  5. 5Click Activate to enable the Flow
  6. 6Click Save

  • ✅Make edits in Account Editor during the grace period
  • ✅After the wait, confirm payment is automatically captured
  • ✅Confirm the processed_by_AE tag appears on the order
  • ✅Confirm the order appears in your 3PL with all edits included
  • ✅Place a test POS or draft order — verify it captures immediately (no wait, no tag)
⚠️ Troubleshooting
Issue Likely Cause Fix
Orders capture immediately (no wait) Payment capture set to “Automatically at checkout” Change to “Manually capture payment for orders” in Shopify Settings → Payments
Flow not triggering at all Flow is not activated or was deactivated by a Shopify update Open Shopify Flow → check the Flow is Active → re-activate if needed
Orders stuck in “Authorized” forever Flow errored during execution Check Flow run history for errors → re-import the Flow if needed
Tag processed_by_AE not appearing Order captured via an immediate branch (POS, high risk, restricted) This is expected — only orders that go through the full wait get tagged
Online Store orders capturing too fast Wait time set too short in the Flow Edit the Flow → increase the wait time to match your editing deadline
POS orders waiting instead of capturing immediately Channel condition not set correctly Verify Condition 1 checks for “Online Store” channel name exactly
