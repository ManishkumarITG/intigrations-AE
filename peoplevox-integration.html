Peoplevox Integration — Shopify Edited Order Sync (Account Editor)

3PL Integrations  •  Last updated: April 2026

This guide helps merchants using Peoplevox (WMS) understand how to configure the integration so that Account Editor order edits are always reflected correctly. Peoplevox reads live edits based on the iPaaS (Integration Platform as a Service) associated with it — however, the delayed capture flow is still required to prevent premature order downloads.

📝
iPaaS-dependent live edits: Peoplevox can read live edits depending on your iPaaS configuration. However, the Account Editor Delayed Master Flow is still required to delay the initial download — ensuring Peoplevox does not import the order before the editing window closes. Contact your iPaaS provider and Peoplevox representative to confirm whether live edit syncing is active on your account.
🔄 How the Integration Works

Peoplevox downloads orders from Shopify only when payment status is “Paid”. The Account Editor Delayed Master Flow creates a grace period between order creation and payment capture — giving you time to edit the order in Account Editor before Peoplevox ever sees it.

1Order Created

Customer places an order → Payment is Authorized (not yet captured).

2Grace Period Active

The editing window opens. Customer or merchant can edit the order in Account Editor — add items, remove items, change quantities, update address.

3Peoplevox Skips the Order

Because payment status is still Authorized, Peoplevox does not download the order. The order is invisible to Peoplevox during the entire editing window.

4Grace Period Ends

Shopify Flow automatically captures payment → Status updates to Paid.

5Peoplevox Downloads

Peoplevox sees the order for the first time with all edits already included — correct items, quantities, and totals.

🪜 Step-by-Step Setup
📋 Step 1 — Enable Manual Payment Capture in Shopify
✅ Step-by-Step Configuration

Set Shopify to hold payment in Authorized state so Peoplevox cannot import the order until editing is complete and payment is captured.

  1. 1Log in to Shopify Admin
  2. 2Go to Settings → Payments
  3. 3Scroll down to Payment capture method
  4. 4Select “Manually capture payment for orders”
  5. 5Click Save
⚠️
7-day limit: Shopify Payments authorizations expire after 7 days. Always capture payment within this window after editing is complete.
⚡ Step 2 — Import the Account Editor Delayed Master Flow
✅ Step-by-Step Configuration

The Account Editor Delayed Master Flow is a pre-built Shopify Flow automation that holds payment capture for a configurable grace period, then captures automatically — triggering Peoplevox to download the finalized order.

  1. 1Download the Account Editor Delayed Master Flow file (provided with this guide)
  2. 2In Shopify Admin, go to Apps → Shopify Flow
  3. 3Click Import and select the downloaded .flow file
  4. 4Configure the grace period timing to match your editing deadline (e.g., 30 minutes)
  5. 5Click Activate to enable the Flow
  6. 6Click Save
📥 Download Account Editor Delayed Master Flow

📝
How the Flow works: When a new order is created, the Flow waits for the configured grace period. After the wait, it automatically captures payment — changing the status from Authorized to Paid. Peoplevox then downloads the order with all edits included.
🚫 Step 3 — Disable Shipping Methods App Block in Account Editor
⚠️ Required — Peoplevox Limitation

Peoplevox does not support shipping method changes during the editing window. To prevent customer confusion, you must disable the Shipping Methods App Block in Account Editor settings.

  1. 1Open your Account Editor app settings in Shopify Admin
  2. 2Navigate to the App Blocks configuration
  3. 3Find the Shipping Methods App Block
  4. 4Set it to Disabled
  5. 5Click Save
🛑
Do not skip this step. If the Shipping Methods App Block is left enabled, customers may attempt to change their shipping method during the editing window. Peoplevox will not reflect these changes, leading to incorrect shipping and customer complaints.
⏱️ Step 4 — Configure the Editing Deadline
✅ Step-by-Step Configuration

Set how long customers have to edit their order before the grace period expires and payment is captured. This deadline should match the timing configured in the Shopify Flow.

  1. 1Open Account Editor app settings
  2. 2Find the Editing Deadline setting
  3. 3Set the deadline to match your Flow timing (e.g., 30 minutes)
  4. 4Click Save
📝
Keep timings in sync: The editing deadline shown to the customer should match (or be slightly shorter than) the grace period in Shopify Flow. For example, if the Flow waits 30 minutes before capturing, set the editing deadline to 25–30 minutes so customers see an accurate countdown.
✏️ Step 5 — Your Account Editor Workflow
✅ How Every Order Works After Setup

Once setup is complete, this is the automated flow for every order:

  1. 1New order arrives → Payment = Authorized (Peoplevox does not see it ✅)
  2. 2Grace period active → Customer can edit in Account Editor — add/remove items, change quantities, update address
  3. 3Grace period expires → Shopify Flow automatically captures payment
  4. 4Payment becomes Paid → Peoplevox downloads the finalized order ✅
  5. 5Order is shipped with all edits included — no manual intervention needed

🧪 Testing Checklist

Before going live, run through this checklist to confirm everything works correctly:

  • ✅Place a test order — verify it does not appear in Peoplevox immediately
  • ✅Make edits during the editing window — add/remove items, change address
  • ✅Wait for the grace period to expire
  • ✅Confirm payment is captured automatically by the Flow
  • ✅Confirm order appears in Peoplevox with all edits included
  • ✅Verify shipping method changes are blocked (App Block disabled)
⚠️ Troubleshooting
Issue Likely Cause Fix
Orders appearing in Peoplevox too early Payment capture not set to Manual, or Flow not activated Verify payment capture = “Manually capture payment for orders” and the Account Editor Delayed Master Flow is Active
Orders not appearing after grace period Flow is not capturing payment correctly Open Shopify Flow → check the run history for errors → re-import if needed
Edits not reflected in Peoplevox Edits made after grace period, or iPaaS not configured for live sync Ensure edits are completed before the grace period expires. If live edit sync is expected, confirm your iPaaS integration is configured properly with Peoplevox
Customer changed shipping method Shipping Methods App Block is still enabled Disable the Shipping Methods App Block in Account Editor settings
Authorization expired (no payment captured) Grace period exceeds 7-day limit Reduce grace period to well under 7 days (recommended: 30 min – 2 hours)
🧠 Best Practices
  • Keep the grace period short — 30 minutes to 2 hours is ideal for most stores
  • Always disable the Shipping Methods App Block — Peoplevox cannot process shipping changes
  • Sync the editing deadline in Account Editor with the Flow timing so customers see an accurate countdown
  • Monitor Shopify Flow run history after launch to confirm payment captures are firing correctly
  • Test the full flow end-to-end before going live with real orders
